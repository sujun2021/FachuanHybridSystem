# v26.51.8 - 2026-06-07

## 后端

### 新功能

- **CloakBrowser 反检测浏览器全面替换**：所有浏览器启动路径统一从原生 Playwright 切换到 CloakBrowser（Chromium 146 + 58 个 C++ 源码级补丁），消除 `navigator.webdriver`、Canvas 指纹、WebRTC 泄露等检测信号
  - `launcher.py`：同步启动路径替换为 `cloakbrowser.launch()`
  - `cdp_connector.py`：CDP 路径替换为 `cloakbrowser.launch_async()`，修复了之前 CDP 模式完全不应用反检测的缺陷
  - `__init__.py`：异步原生路径替换为 `cloakbrowser.launch_async()`
  - `service.py`：`BrowserService._get_or_create_browser()` 适配 CloakBrowser，补充 `IBrowserService` 协议兼容方法（`get_browser`、`close_browser`、`create_context`）
  - `anti_detection.py`：大幅简化，删除 UA 池、JS 注入脚本和 `playwright-stealth` 依赖，只保留 viewport/locale/timezone 配置
  - `profiles.py`：移除 `--disable-blink-features=AutomationControlled` 和 `--no-sandbox`（CloakBrowser 内置）
  - `Dockerfile`：`playwright install chromium` 替换为 `cloakbrowser install`
  - `pyproject.toml`：`playwright-stealth` 替换为 `cloakbrowser>=0.3.31`
- **macOS 指纹补充补丁**：CloakBrowser macOS 只有 26 个 C++ 补丁（vs Linux 58 个），通过 `page.add_init_script()` 补充 navigator 属性、屏幕尺寸和 Canvas 指纹随机化
  - `anti_detection.py`：新增 `apply_macos_patches(page)` / `apply_macos_patches_async(page)`，仅在 macOS 上生效
  - 三个启动路径（launcher、cdp_connector、__init__）均在创建 page 后调用
- **OneDrive/Dropbox OAuth 授权持久化**：device_code 存入数据库，解决 `runserver` auto-reload 杀死后台轮询线程导致授权丢失的问题
  - `models.py`：新增 `onedrive_pending_device_code`、`onedrive_pending_expires_at`、`dropbox_pending_device_code`、`dropbox_pending_expires_at` 4 个字段
  - `admin.py`：发起授权时持久化 device_code，轮询成功/失败/超时时清除；新增 `resume_pending_device_code_polls()` 进程启动恢复函数
  - `apps.py`：`CoreConfig.ready()` 中调用恢复函数
  - `admin.py` changeform_view：从数据库读取 pending 状态（不仅从内存），进程重启后页面仍显示"授权进行中"

### 修复

- **扫描子文件夹越界误报**：当 `folder_path = "/"` 时，`str(root) + "/"` 变成 `"//"`，导致 `"/子目录".startswith("//")` 为 False 误报越界。修复为特殊处理 root="/" 的情况
  - `cases/services/material/folder_scan_service.py`
  - `contracts/services/contract/integrations/folder_scan_service.py`
- **S3 `get_file_info` 目录检测**：S3 没有真正的目录，`head_object("mydir")` 返回 404。修复为三级检测：文件 → 目录标记(trailing-slash key) → 虚拟目录(子对象前缀)
  - `s3_provider.py`
- **GDrive 共享云盘支持**：所有 API 调用添加 `supportsAllDrives=True, includeItemsFromAllDrives=True`，共享云盘文件不再被静默排除
  - `gdrive_provider.py`
- **GDrive 限流重试**：新增 `_execute_with_retry()` 指数退避函数，处理 429/5xx 错误（最多 5 次，最大 32 秒）
  - `gdrive_provider.py`
- **GDrive 上传优化**：`MediaIoBaseUpload` 添加 `resumable=True`（可恢复上传），MIME 类型从硬编码 `application/octet-stream` 改为 `mimetypes.guess_type()` 自动检测
  - `gdrive_provider.py`
- **Dropbox OAuth HTTP 状态码检查**：`complete_device_code_flow` 添加 `resp.raise_for_status()`，5xx 错误不再被静默忽略
  - `dropbox_provider.py`
- **Dropbox mkdir 错误处理**：从捕获所有 `ApiError` 改为只捕获 `path/conflict`（目录已存在），权限错误等正常向上传播
  - `dropbox_provider.py`

### 优化

- **浏览器架构去重**：删除已废弃的 `scraper/core/browser_service.py`、`browser_manager.py`、`anti_detection.py` 和 `scraper/config/browser_config.py`，所有浏览器操作统一走 `apps/core/services/browser/`
  - 修改 4 个调用方（`automation_token.py`、`auto_login_service.py`、`document_delivery_service.py`、`document_delivery_playwright_service.py`）
  - `base.py`、`court_document/main.py`：删除 3 处冗余的 `inject_stealth_script(page)` 调用（context 级别已有 stealth）
  - `test_service.py`：迁移为 `create_browser("court_zxfw")`
- **LibreOffice 路径查找去重**：3 处重复实现合并为 1 个 `apps/core/services/libreoffice.py`，合并 macOS `~/Applications`、`program/soffice` 路径
  - 删除 `engine.py`、`pdf_merge_utils.py`、`file_prepare_service.py` 中的本地实现
- **错误码集中管理**：新建 `apps/core/exceptions/error_codes.py`，收录 14 个跨模块共享错误码，替换 25 个文件中的裸字符串
  - `TEMPLATE_RENDER_ERROR`（8 个文件）、`PDF_MERGE_FAILED`、`SMS_SUBMIT_FAILED` 等 13 组重复
- **临时文件泄漏修复**：修复 9 处 `mkdtemp` 从不清理的泄漏
  - `document_delivery` 模块 7 处：`_downloading_mixin.py`、`document_delivery_service.py`、`_process.py` 等添加 `finally: shutil.rmtree()`
  - `chrome_process.py`：`kill_chrome()` 添加 `_cleanup_temp_user_data_dir()` 清理临时用户数据目录
  - `pdf_text_extractor.py`：`pdf_to_images()` 返回 `(paths, tmp_dir)` 元组，调用方添加清理
