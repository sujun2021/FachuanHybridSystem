# v26.52.13 (2026-06-17)

## 新增：LLM 隐私审查 CI 步骤

现有 `security_guard.py` 使用正则检测结构化 PII（身份证号、手机号、邮箱、token），但无法识别自然语言中的真实姓名、公司名、案号等上下文敏感信息。

---

## 问题背景

changelog 中曾泄露真实公司名和人名，正则规则无法覆盖此类场景，需要引入大模型做上下文感知的隐私审查。

---

## 实现方案

新增 `privacy_review.py` 脚本，调用 Claude API 对变更文件的新增内容做隐私审查：

| 组件 | 说明 |
|------|------|
| `backend/devtools/privacy_review.py` | 审查脚本，读取 diff 新增行，发送给 Claude 分析 |
| `.github/workflows/backend-ci.yml` | 新增 `privacy-review` job，与 `security-guard` 并行 |

**检测范围：**
- 真实人名（非占位符）
- 真实公司/组织名称
- 法院案号
- 真实地址
- 身份证号、手机号、银行卡号等

**容错设计：**
- `ANTHROPIC_API_KEY` 未设置时跳过（不影响无 key 环境）
- LLM 调用失败时打印警告并跳过（不阻塞 CI）
- 使用 Haiku 模型，控制成本

---

## 配置要求

需在 GitHub repo Settings → Secrets → Actions 中添加 `ANTHROPIC_API_KEY`。
