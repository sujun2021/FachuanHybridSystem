# v26.52.3 (2026-06-13)

## 工作流引擎 + Evidence 模块重构

本版本包含三大块改动：**DynamicWorkflow 通用工作流引擎上线**、**工作流前端体验优化**、**Evidence 模块架构整理**。

---

## 一、DynamicWorkflow 通用动态工作流引擎

新增基于 Temporal 的通用工作流引擎，支持可视化编排、AI Agent 通过 MCP 工具直接创建工作流。

### 核心改动

| 改动 | 说明 |
|------|------|
| **DynamicWorkflow 引擎** | 基于 Temporal 的通用动态工作流，支持审批、节点执行、异步调度 |
| **工作流模板可视化编排器** | 前端拖拽式模板编辑器，支持节点连线、配置 |
| **MCP 工具集成** | 新增模板管理 MCP 工具，AI Agent 可直接创建和管理工作流模板 |
| **工作流运行删除** | 支持删除已完成的工作流运行记录 |
| **端到端集成测试** | DynamicWorkflow 完整集成测试覆盖 |

### API 变更

| 变更 | 说明 |
|------|------|
| 合并模板 API 到 `workflow_api` | 修复独立路由 404 问题 |
| 移除 API 路径尾斜杠 | 匹配 `ApiTrailingSlashMiddleware` |
| 审批/取消接口 | 添加错误处理 |
| 注册 `/admin/workflows` 路由 | 修复后台 404 |

## 三、Evidence 模块架构整理

将证据管理的 admin 和 model 彻底归口到 `evidence` app，清理历史遗留的死代码和误导性模块。

### 核心改动

| 改动 | 说明 |
|------|------|
| **创建 Proxy Model** | `EvidenceListProxy` / `EvidenceItemProxy`（`app_label="evidence"`），admin 从 `/admin/documents/` 移到 `/admin/evidence/` |
| **删除 HearingNote** | 彻底移除庭审笔记模型、admin、测试和关联引用 |
| **删除 documents/admin/evidence/ 死代码** | 旧版 evidence admin 副本（~900 行），从未被实际使用 |
| **修正「案件材料整理」名称** | 其他工具页中 `evidence_sorting` 显示名从「证据整理」改为「案件材料整理」 |

### Admin 变更

```
变更前:                              变更后:
/admin/documents/                    /admin/evidence/
  ├── evidencelist/  证据清单           ├── evidencelistproxy/  证据清单
  └── evidenceitem/  证据条目           ├── evidenceitemproxy/  证据条目
                                       └── evidencegroup/      争议焦点分组
/admin/evidence/
  ├── evidencegroup/  争议焦点分组
  └── hearingnote/  庭审笔记  ← 已删除
```

### 迁移文件

| 迁移 | 说明 |
|------|------|
| `evidence/0002_delete_hearingnote` | 删除 HearingNote 表 |
| `evidence/0003_add_proxy_models` | 创建 EvidenceListProxy / EvidenceItemProxy |
