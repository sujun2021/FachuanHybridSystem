# v26.52.2 (2026-06-12)

## Refactor: 彻底移除硅基流动（SiliconFlow）后端

删除 SiliconFlow LLM 后端实现，统一为 OpenAI-compatible + Ollama 双后端架构。修复 `achat`/`astream` async fallback bug，清理全部配置、代码、测试、数据库残留。

### 背景

SiliconFlow 后端已长期不可用（401 Unauthorized），且其 `SiliconFlowBackend` 类被 `OpenAICompatibleBackend` 继承（仅共享 ~70 行工具代码）。移除后架构更清晰，async 调用不再错误回退到 `SILICONFLOW_*` 配置。

### 核心改动

| 改动 | 说明 |
|------|------|
| **删除 `siliconflow.py`** | 374 行后端实现彻底删除 |
| **`OpenAICompatibleBackend` 独立化** | 不再继承 `SiliconFlowBackend`，内移 `__init__`、`_normalize_messages`、`_extract_usage`、`_extract_content`、`embed_texts`、`is_available` 等方法 |
| **修复 async fallback bug** | `achat`/`astream` 在 `_config=None` 时原错误调用 `LLMConfig.get_default_model_async()`（读 `SILICONFLOW_DEFAULT_MODEL`），现改为 `LLMConfig.get_openai_compatible_model_async()` |
| **默认后端变更** | `"siliconflow"` → `"openai_compatible"` |
| **新增 `get_openai_compatible_model_async()`** | 补充缺失的 async 配置方法 |
