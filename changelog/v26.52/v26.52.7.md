# v26.52.7 (2026-06-14)

## 身份证合并页面接入 ONNX 方向自动检测

在图片旋转工具的身份证合并页面中集成 ONNX 方向检测模型，实现上传图片后自动检测并校正方向，同时新增历史任务管理功能。

---

## 一、ONNX 自动检测集成

| 功能 | 说明 |
|------|------|
| 自动方向检测 | 上传图片后自动调用 ONNX 模型检测方向，无需手动旋转 |
| 批量检测支持 | 支持多张图片并行检测，~1秒/张 |
| 检测结果显示 | 检测完成后显示置信度和耗时统计 |
| OCR 提示集成 | 检测完成后提示用户是否提取文字并重命名 |

---

## 二、历史任务管理

| 功能 | 说明 |
|------|------|
| 任务保存 | 支持将当前编辑的任务保存为历史记录 |
| 任务列表 | 分页展示历史任务，支持搜索和刷新 |
| 任务详情 | 查看历史任务的所有页面和旋转状态 |
| 任务重命名 | 支持修改历史任务名称 |
| 任务删除 | 支持删除历史任务（二次确认） |
| 导出 URL 保存 | 导出的 ZIP/PDF 链接自动关联到历史任务 |

---

## 三、数据模型新增

| 模型 | 说明 |
|------|------|
| `ImageRotationJob` | 历史任务主表，存储任务名称、状态、导出链接 |
| `ImageRotationJobPage` | 任务页面表，存储源图、旋转角度、OCR 结果 |
| `onnx_rotation` 字段 | 新增字段，分离 ONNX 自动旋转与手动调整量 |

### 字段设计

```
detected_rotation: 手动调整量（delta）
onnx_rotation: ONNX 自动旋转角度
总旋转角度 = onnx_rotation + detected_rotation
```

## 五、前端交互优化

| 优化 | 说明 |
|------|------|
| 自动保存 | ONNX 检测完成后自动保存到历史 |
| 变更同步 | 手动旋转后 800ms 自动同步到后端 |
| 旋转分离 | 前端 `rotation` 存手动调整量，`autoRotation` 存 ONNX 角度 |
| CSS Transform | 历史视图使用 `(onnx_rotation + detected_rotation) % 360` 显示正确旋转 |

---

## 六、Bug 修复

| 修复 | 说明 |
|------|------|
| 历史任务旋转角度不更新 | `syncChangesToBackend` 在首次保存时正确等待 `saveTaskToHistory` 完成 |
| 历史视图旋转显示错误 | CSS 三元表达式在 `onnx_rotation=0` 时正确应用 `detected_rotation` |
| 导出旋转角度计算 | ZIP/PDF 导出时正确计算总旋转角度 |
| 历史加载到编辑器 | `rotation` 字段直接使用 `detected_rotation`，避免双重旋转 |
