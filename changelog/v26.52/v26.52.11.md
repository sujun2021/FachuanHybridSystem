## 一、修复：验证码识别器支持无 task 场景

**问题：** 消息中心定时同步法院收件箱时，抛出 `RuntimeError: 验证码识别需要 task 参数`

**原因：** 消息中心同步场景没有 `ScraperTask` 对象，`get_captcha_recognizer()` 在插件不可用且无 task 时直接抛出异常。

**修复：**

### 新增 `FileBasedCaptchaRecognizer`

不依赖 `ScraperTask`，基于文件系统等待手动输入：

- 保存验证码图片到 `{MEDIA_ROOT}/automation/captcha_pending/{uuid}.png`
- 创建 `{uuid}.answer` 空文件
- 轮询等待 `.answer` 文件中出现内容
- 用户写入答案后返回

### 改进 `get_captcha_recognizer()` 降级逻辑

```
优先级：
1. captcha_ocr 插件已安装且 ddddocr 可用 → DdddocrRecognizer（自动识别）
2. 有 task 参数 → ManualCaptchaRecognizer（手动输入，依赖 task 状态）
3. 无 task 且插件不可用 → FileBasedCaptchaRecognizer（基于文件的手动输入）
```

**不再抛出异常，始终返回可用的识别器。**

## 二、修复：依赖缺失问题

### 2.1 添加 onnxruntime

**问题：** `ImportError: onnxruntime is not installed`

**原因：** RapidOCR 3.x 需要单独安装推理引擎（onnxruntime/openvino/torch），但 pyproject.toml 未声明。

**修复：** 添加 `onnxruntime>=1.25.0` 到 pyproject.toml

### 2.2 恢复 ddddocr

**问题：** `ddddocr 未安装，请运行: uv add ddddocr`

**原因：** commit `76c854c6e` 移除 ddddocr 时，onnxruntime 作为间接依赖也被清除。

**修复：** 添加 `ddddocr>=1.6.0` 到 pyproject.toml
