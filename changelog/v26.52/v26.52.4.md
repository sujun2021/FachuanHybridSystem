# v26.52.4 (2026-06-13)

## ONNX 方向检测集成

本版本将图片方向检测从纯 OCR 投票方案切换为 **ONNX ConvNeXtV2-Atto 分类器**，大幅提升检测速度和准确率。

---

## 一、ONNX 方向检测服务

新增基于 Fachuan 微调 ConvNeXtV2-Atto 的 ONNX 方向分类器，支持本地推理。

| 改动 | 说明 |
|------|------|
| **ONNX 推理引擎** | 基于 onnxruntime 的 4 分类器（0°/90°/180°/270°），EXIF 转正后运行 |
| **HuggingFace Hub 自动下载** | 本地无模型时自动从 `Fachuan/orientation-classifier` 下载 |
| **置信度阈值 0.65** | 4 分类随机基线 0.25，0.65 以上才自动旋转，避免误判 |
| **PIL rotate() 统一** | 训练用 `rotate(angle)` 逆时针，推理用 `rotate(-angle)` 顺时针复原 |

---

## 二、API 层改动

| 改动 | 说明 |
|------|------|
| `detect_orientation` 接口支持 `method` 参数 | `"onnx"`（默认）或 `"ocr_voting"`，可按需切换检测方式 |
| 新增 `/extract-text` 接口 | 独立的文字提取接口，支持 `local` / `paddleocr_api` 两种 provider |
| `can_auto_rotate` 返回字段 | 仅在置信度 ≥0.65 且旋转角度 ≠0 时返回 true |
| **推理计时** | 每张图片返回 `elapsed_ms`，批量接口返回 `total_elapsed_ms` |

## 三、前端改动

| 改动 | 说明 |
|------|------|
| **统计栏新增 ONNX 耗时** | 检测完成后在 stats-bar 显示总推理耗时 |
| **图片卡片推理耗时** | 每张图片右下角显示单张推理时间 |
| **OCR 流程分离** | ONNX 检测完成后弹出 OCR 提取文字提示框，用户可选本地/远端 OCR 或跳过 |
| 新增 `showOcrPrompt` / `totalDetectionTimeMs` 状态 | 管理 OCR 提示显示和计时数据 |

---

## 四、Bug 修复

| 修复 | 说明 |
|------|------|
| EXIF 方向标签双重旋转 | 推理前先 `ImageOps.exif_transpose()`，避免 ONNX 输出与 EXIF 叠加 |
| 低置信度误判导致正确图片被旋转 | 引入 `can_auto_rotate` 标志，低于阈值不执行旋转 |
| 180° 翻转误判 | 增加宽高比启发式辅助判断 |
| 微信支付截图无法自动旋转 | 置信度阈值从 0.8 降至 0.65 |
