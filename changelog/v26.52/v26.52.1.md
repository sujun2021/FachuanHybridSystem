# v26.52.1 (2026-06-12)

## Feat: Temporal Workflow Engine — 诉讼工作流自动化基础设施

集成 Temporal Python SDK，实现持久化诉讼工作流引擎。支持多步骤流程编排、人工审批 Gate、外部事件驱动恢复、全链路审计。

### 架构

```
Claude Desktop/Code  →  MCP Server (376 tools + 5 workflow tools)  →  Django API
                                                                  ↓
前端 Dashboard  →  REST API (5 endpoints)  →  WorkflowRun / StepExecution (Django ORM)
                                                                  ↓
                                              Temporal Worker (Python)  →  Temporal Server
```

## Fix: LLMService async 兼容性

修复 Temporal Activity 在 async 上下文中调用 `LLMService` 时的 `SynchronousOnlyOperation` 错误。

### 问题根因

`LLMConfig._get_system_config()` 是同步方法，内部调用 `SystemConfigService.get_value()` 查询 Django DB。在 Temporal Activity 的 async 上下文中触发 Django 的 `SynchronousOnlyOperation`。

### 修复方案

| 文件 | 改动 |
|------|------|
| `llm/config.py` | 新增 `_config_cache` 类变量（跨调用缓存）；`_get_system_config()` 检测 async 上下文时 fallback 到 Django settings；`_get_system_config_async()` 写入缓存；新增 `get_default_backend_async()` |
| `llm/service.py` | 新增 `LLMService.create()` async 工厂方法 — 预热所有后端配置缓存（api_key、base_url），后续同步调用命中缓存 |
| `activities.py` | 新增 `_llm_chat()` 辅助函数 — 统一 LLM 初始化 + 消息构建 + 调用；所有 LLM Activity 改用 `LLMService.create()` + `llm.achat()` |
