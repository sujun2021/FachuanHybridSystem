# v26.52.12 (2026-06-17)

## 修复：归档PDF页码在旋转页面(rotation=270)上显示异常

合同 202 生成归档文件夹时，"4-案卷材料.pdf" 第 31-40 页（/Rotate=270 的横向页面）页码只显示个位数，且位置和方向错误。

---

## 问题描述

**现象：** 归档文件夹中"4-案卷材料（广东顺德菊花湾农业发展有限公司诉徐胜合同纠纷一案）_20260617.pdf"的第 31-40 页页码异常：
- 本应完整显示每页页码（31/32/.../40），实际只显示个位数（1/2/.../0）
- 第 41 页之后恢复正常

**根因：** 原 `add_page_numbers()` 函数使用 `page.insert_text()` 直接在内容坐标系写入文字，未处理页面旋转：
- `/Rotate=270` 的页面，内容坐标系与视觉坐标系存在 90° 旋转偏差
- 直接写入的文字被页面旋转带动，导致方向错误、位置偏移

---

## 修复方案

使用 PyMuPDF 官方推荐的 `derotation_matrix` 方案（参考 `PyMuPDF/discussions/3366`）：

| API | 作用 |
|-----|------|
| `page.derotation_matrix` | 视觉坐标 → 内容坐标（自动处理旋转） |
| `insert_text(rotate=rotation)` | 自动旋转文字匹配页面旋转 |
| `draw_rect(overlay=True)` | 白色背景矩形遮盖源 PDF 原有页码（KSPX） |

**核心逻辑：**
```python
# 有旋转的页面：先转换坐标，再带旋转写入
derot = page.derotation_matrix
text_point = fitz.Point(x, y) * derot
page.insert_text(text_point, text, rotate=page.rotation, overlay=True)
```

**影响文件：**
- `apps/contracts/services/archive/generation/pdf_utils.py` — `add_page_numbers()` 函数重写
