# v26.54.5

## 新功能

### 收件箱提交到短信处理

收件箱详情页新增「提交到短信处理」按钮，支持将邮件/YZW收件箱中的 PDF 附件直接作为文书进入短信处理流程：

- 点击按钮后自动创建 CourtSms 记录，从主题正则提取案号
- 跳过短信解析（PARSING）和下载（DOWNLOADING）阶段，直接进入案件匹配（MATCHING）
- 匹配成功后自动归档到案件绑定目录，发送群聊通知

**修复的关键问题：**
- `body_preview` 方法中 `import html` 位于条件分支内，走 else 分支时引发 `UnboundLocalError`
- 案号提取：Ollama 不可用时自动降级为正则提取（支持全角括号）
- 案号提取：使用统一 LLM 服务（`openai_compatible` 优先），不再硬编码 `backend="ollama"`
- 从收件箱提交时，`document_file_paths` 中的文件能正确归档到案件文件夹（之前因无 `scraper_task` 直接跳过）

## 开发体验改进

- `make run-dev` 现在监控 `backend/plugins` 目录变更，修改 plugins 代码后自动重启
- `manage.py` 新增 `--watch` 选项（仅 `runserver` 命令生效）

## CI 修复

- `settings.py` 的 `_DEV_SECRET_KEY` 添加 `# pragma: allowlist secret`，解决 detect-secrets 误报
- 更新 `pip-audit` ignore 列表，添加 7 个新发现的依赖漏洞 ID（同步 `ci-local.sh` 和 `backend-ci.yml`）
