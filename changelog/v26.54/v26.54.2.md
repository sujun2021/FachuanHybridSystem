# v26.54.2

## 新增功能

### OA 半自动操作（打开OA / 申请开票 / 申请所函盖章）

合同详情页和案件详情页新增三个按钮，点击后自动完成前置步骤（登录→搜索→填表），有头浏览器停在目标页面等待用户手动操作：

- **归档标签 → 打开OA**：打开 OA 归档页面，输入案件编号、填写案件小结、清除默认附件
- **收费与财务标签 → 申请开票**：打开发票管理页面，输入案件编号、搜索、跳转到开票页面
- **文档生成标签 → 申请所函盖章**：打开盖章文书管理页面，搜索案件、填写表单（所函/公章+电子公章/3份）

API 端点：
- `POST /api/v1/oa-archive/open-oa`
- `POST /api/v1/oa-archive/open-invoice`
- `POST /api/v1/oa-archive/open-stamp`

### 文档解析后台任务回写

修复 `DocumentParsingTask` 上传后永远停在「处理中」的问题。新增 `document_parsing_hook()`，Django-Q 任务完成后自动将解析结果回写到模型。

## 重构

### OA Filing 适配器模式重构

将 `ScriptExecutorService`（原 577 行）拆分为：

- **Protocol 定义**（`base_firm_adapter.py`）：FilingAdapter / StampAdapter / ArchiveAdapter / CaseImportAdapter / ClientImportAdapter
- **注册表**（`oa_firm_registry.py`）：按 `site_name` 查找 adapter，新增律所只需注册一行
- **JTN 适配器**（`jtn/adapter.py`，438 行）：实现所有 Protocol，只做薄委托 + 字段映射
- **通用调度器**（`script_executor_service.py`，298 行）：session 管理 + 凭证查找 + 线程调度，零律所特有逻辑

### 消除 JTN 硬编码

- `site_name` 全部改为参数传入（默认值 `"金诚同达OA"` 仅为兼容）
- `get_jtn_credential()` → `get_credential(lawyer_id, site_name)`
- `SUPPORTED_SITES` 改由 `oa_firm_registry` 导出
- 案件/客户导入服务改用 adapter 交互 OA，不再直接 import jtn 模块

### 通用数据类抽取

`OACaseData` / `OACustomerData` 等数据类从 `jtn/models.py` 迁移到 `oa_data_models.py`，JTN 模块 re-export 保持兼容。

### Playwright 逻辑回归脚本模块

`open_oa_page` / `open_invoice_page` / `open_stamp_page` 的 Playwright 自动化逻辑从 adapter 移回各功能模块的 `playwright_*.py`（mixin），adapter 只做薄委托。新建 `jtn/invoice/` 模块。

### jtn/ 目录清理

- `html_parser.py` → 移入 `case_import/`（唯一消费者）
- `jtn/models.py` 中 JTN 特有模型 → 移入 `case_import/models.py`
- 删除 `jtn/models.py`
