# v26.54.1 (2026-07-07)

## 新增功能

### OA 归档材料自动提交

新增 JTN OA 结案归档材料自动提交功能，复用现有登录和案件搜索流程，通过 Playwright 自动化完成归档文件上传。

**API 端点**（`/oa-archive`）：
- `POST /lookup` — 根据文件路径反查合同，返回 OA 案件编号
- `POST /apply` — 发起归档提交（异步执行，返回 session 供轮询）
- `GET /session/{id}` — 查询归档提交状态

**自动化流程**：
1. 登录 OA（缓存 cookies → SSO 扫码）
2. 导航到结案归档管理页面（`projcloseapp.aspx`）
3. 通过 layui 弹窗搜索并选择案件
4. 填写案件小结（默认"详见卷宗"）
5. 上传归档文件到"案件业务卷宗"行
6. 点击保存提交

**新增文件**（16 个）：
- `models/archive_session.py` — `ArchiveSession` 模型（`file_paths` 用 `JSONField` 支持多文件）
- `schemas/archive_schemas.py` — 输入/输出 schema
- `api/archive_api.py` — 3 个 API 端点
- `admin/archive_session_admin.py` — Django Admin 注册
- `services/oa_scripts/jtn/archive/` — Playwright 自动化模块（constants、models、playwright、service）
- `services/script_executor_service.py` — 新增 `execute_archive` 线程调度
- `migrations/0014_archive_session.py` — 数据库迁移

## 技术细节

- 归档文件上传使用 `#tblFiles input[type=file]` 选择器，跳过页面顶层隐藏的 file input
- 案件小结 `#proje_report1` 为 readonly textarea，通过 JS 去除 readonly 后赋值
- 搜索弹窗复用盖章模块的 `searchProject` iframe 模式
- `ArchiveSession.file_paths` 使用 `JSONField` 存储多文件路径列表
